DeepML19_Principal Component Analysis (PCA)

  • PCA利用特征值和特征向量来识别数据集的主成分
  • 对于给定的矩阵AAA，表示PCA中的协方差矩阵，特征值λ\lambdaλ及其对应的特征向量vvv满足：Av=λvAv=\lambda vAv=λv
  • 矩阵的特征值AAA通过求解特征方程可以得出det(A−λI)=0det(A-\lambda I)=0det(A−λI)=0，III是与AAA维度相同的单位矩阵
  • 特征值的大小与其对应的特征向量在表示数据集变异性方面的重要性相关，通过选择与最大特征值相对应的特征向量子集，PCA可以实现降维，同时尽可能多地保留数据集的变异性
  • 实际应用
    • 标准化数据集：保证每个特征的平均值是000，标准差是111
    • 计算协方差矩阵：反映特征如何一起变化
    • 寻找特征值和特征向量：求解协方差矩阵的特征方程
    • 选择主成分：选择具有最高特征值的特征向量（成分）进行降维

Neo4j学习

安装过程

  1. 需要已经安装好homebrew和JAVA
  2. 终端输入brew install neo4j
  3. 进入neo4j官网：选择products-选择deploy-选择deployment center，链接
  4. 选择对应版本下载得到tar.gz文件，解压
  5. 进入Downloads文件夹，改名：`mv neo4j-community-5.1.0 NEO4J_HOME
  6. 将文件移动到home目录，cp NEO4J_HOME $HOME
  7. 启动，终端输入：neo4j start
  8. 停止，终端输入：neo4j stop
  9. 安装插件：下载对应的文件放入plugins文件夹中

运行过程

  1. 可以下载其他语言的驱动，导入数据
  2. 数据都是共享的，需要删除数据则：
    • 删除所有节点：
      • MATCH (n)
      • DETACH DELETE n;
    • 删除所有关系：
      • MATCH ()-[r]-()
      • DETACH DELETE r;
    • 删除所有属性：
      • MATCH (n)
      • DETACH DELETE n;
    • 删除标签：得一个个删：
      • 查看索引：:schema
      • 删除对应的：DROP CONSTRAINT constraint_4a7831e6

LeetCode31_下一个排列

如何得到

  1. 希望下一个数比当前数大，将后面的【大数】和前面的【小数】交换得到更大的数，比如 123456 ，将 5 和 6 交换能得到一个更大的数 123465
  2. 希望下一个数增加的幅度尽可能小，这样才满足与当前排列紧邻
    • 在尽可能靠右的低位进行交换 -> 需要从后往前查找
    • 将一个尽可能小的【大数】与前面的【小数】交换
    • 将【大数】换到前面后，要将【大数】后面的所有数重置为升序
  3. 例子，比如 123465 ，交换 5 和 4 ，得到 123564 ；然后将 5 之后的数重置为升序，得到 123546

算法过程

  1. 从后向前查找第一个相邻升序元素对 (i,j) ，满足 A[i] < A[j]，这时 [j,end]必然是降序
  2. 在 [j,end] 从后向前查找第一个满足 A[i] < A[k]的 k，A[i]、A[k]分别是【小数】、【大数】
  3. 将 A[i] 与 A[k] 交换
  4. 这时 [j,end] 必然是降序，逆置 [j,end]，使其升序
  5. 如果在步骤1找不到符合的相邻元素对，说明当前 [begin,end] 为一个将序顺序，则直接跳到步骤4

LeetCode2071_你可以安排的最多任务数目

如果完成了xxx个任务，最佳方案是力量最大的xxx个worker完成了要求力量最小的xxx个任务
按力量从小到大依次考虑每个worker WJW_JWJ​，可以选择的任务是要求力量值不超过WJ+strengthW_J+strengthWJ​+strength且尚未完成的任务之一

  1. 如果当前可选任务中要求力量值最小的一个小于等于WjW_jWj​，则这个worker不用吃药就可以完成任务
  2. 如果没有可以直接完成的，则有药情况下选要求力量值最大的任务比较划算，没药则直接失败
  3. 如果吃药也没能完成，则直接失败

任务按照从小到大的顺序添加到可选任务范围中；移除时要么最小要么最大 --> 双端单调队列

LeetCode402_移掉K位数字

比如 num=1432219,k=3num=1432219, k=3num=1432219,k=3，需要返回一个长度为4的字符串，怎么才能求出这四个位置
前置知识：对于两个数 123a456123a456123a456 和 123b456123b456123b456，如果 a>ba>ba>b ，那么数字 123a456>123b456123a456>123b456123a456>123b456；也就是说两个相同位数的数字大小关系取决于第一个不同数的大小；所以需要前面的数尽量小
思路：保留n−kn-kn−k个元素，对于DiD_iDi​，需要从左往右找到第一个位置iii，使得Di<Di−1D_{i}<D_{i-1}Di​<Di−1​，并删去Di−1D_{i-1}Di−1​，需要删除不超过kkk次

不把栈顶元素弹出的条件：

  • 栈为空
  • 新的栈顶不大于当前数字
  • 已经删除了kkk位数字

额外的处理：

  • 已经删除了mmm个数字并且m<km<km<k，则需要从序列尾部额外删除
  • 最终数字序列存在前导零，要删去前导零
  • 最终数字序列为空，应该返回000
